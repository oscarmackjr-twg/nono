# Quick Task 260513-f5n — Update POC runbook for mandatory BrokerPath

## Problem

`scripts/build-windows-msi.ps1` was updated in Phase 31 Plan 04 to take a new
**Mandatory** `-BrokerPath` parameter pointing at `nono-shell-broker.exe`
(scripts/build-windows-msi.ps1:8-13). The runbook at
`docs/cli/development/windows-poc-handoff.mdx` Step 1 Option C still shows the
pre-Phase-31 invocation, which:

1. Doesn't build the broker for the `x86_64-pc-windows-msvc` target.
2. Doesn't pass `-BrokerPath`, so PowerShell prompts interactively and the
   build cannot complete non-interactively as documented.

A POC operator following the runbook today hits an interactive prompt
(`BrokerPath:`) and stalls. This task fixes the runbook to match the script.

## Scope

Single file: `docs/cli/development/windows-poc-handoff.mdx`.

Two edits:

### Edit 1 — Option C build command (lines 48-58)

Replace the single `cargo build` + MSI invocation with a build step that
produces both the CLI and broker binaries for the MSVC target, then pass
`-BrokerPath` to the MSI script.

Target shape:

```powershell
# Build BOTH binaries for the explicit Windows MSVC target. nono-shell-broker.exe
# is required by the MSI as of Phase 31 Plan 04 — it installs as a sibling of
# nono.exe so the runtime cascade in spawn_windows_child can resolve it via
# current_exe().parent() (D-07).
cargo build --release --target x86_64-pc-windows-msvc -p nono-cli -p nono-shell-broker

# Per-user MSI
.\scripts\build-windows-msi.ps1 `
  -VersionTag v0.37.1-poc.1 `
  -BinaryPath .\target\x86_64-pc-windows-msvc\release\nono.exe `
  -BrokerPath .\target\x86_64-pc-windows-msvc\release\nono-shell-broker.exe `
  -Scope user `
  -OutputDir dist\windows
```

The output line (`Output: dist\windows\nono-v0.37.1-poc.1-...-user.msi`) remains
unchanged — the MSI filename is unaffected by the broker addition.

### Edit 2 — Portable archive snippet (lines 68-72)

Add the broker binary to the `Compress-Archive` `-Path` array so portable-zip
users get the same sibling-deployment shape the MSI does.

Target shape:

```powershell
Compress-Archive `
  -Path .\target\x86_64-pc-windows-msvc\release\nono.exe, `
        .\target\x86_64-pc-windows-msvc\release\nono-shell-broker.exe `
  -DestinationPath .\dist\nono-v0.37.1-poc.1-x86_64-pc-windows-msvc.zip
```

POC users extract to a stable directory; the broker must end up next to
`nono.exe` in that directory or `nono shell` fails at sibling-resolution time
(see Step 4 § Broker.exe Verification at Launch).
