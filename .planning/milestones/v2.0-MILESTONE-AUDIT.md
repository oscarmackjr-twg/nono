# v1.0 Windows Parity Milestone Audit

## Milestone Definition of Done

From `.planning/ROADMAP.md` and `.planning/PROJECT.md`:

> Windows security must be as structurally impossible and feature-complete as Unix platforms, ensuring the dangerous bits are kernel-enforced without compromising the supervisor-led security model.

The milestone (treated as a single bucket spanning v1.0 supervisor/network/state foundations + v2.0 Windows gap closure) requires every requirement in the validated v1.0 list and the active v2.0 list to be implemented, verified, and traced. Concretely:

- v1.0 (Phases 1–6): SUPV-01..05, NETW-01..03, STAT-01/02, DEPL-01/02 — supervisor lifecycle, WFP enforcement activation, snapshots, MSI signing.
- v2.0 (Phases 7–11): WRAP-01, SESS-01/02/03, SHELL-01, PORT-01, PROXY-01, LEARN-01, TRUST-01 (stretch).
