## v1.0 Requirements (Validated in v1.0 Windows Alpha — carried here for traceability)

### Supervisor Control (SUPV)

- [x] **SUPV-01**: User can detach from a running Windows agent using `nono run --detach`. _(Note: v2.0 correction — real flag is `--detached`; runbook typo fixed in Phase 14 plan 14-03.)_
- [x] **SUPV-02**: User can re-attach to a running Windows agent session via Named Pipe IPC.
- [x] **SUPV-03**: User can list all running Windows agent processes using `nono ps`.
- [x] **SUPV-04**: User can atomically stop a Windows agent and its entire process tree using `nono stop`.
- [x] **SUPV-05**: Windows Job Objects are named using session IDs to allow persistent management after CLI exit.

### Network Sandboxing (NETW)

- [x] **NETW-01**: User can block network access for Windows agents using the WFP (Windows Filtering Platform) backend.
- [x] **NETW-02**: WFP rules persist correctly during detach/attach cycles via a background supervisor process.
- [x] **NETW-03**: User can allow specific local ports on Windows via WFP-enforced filtering.

### State Management (STAT)

- [x] **STAT-01**: User can create a filesystem snapshot of a Windows workspace before execution.
- [x] **STAT-02**: User can rollback Windows filesystem changes to a previous snapshot using `nono rollback`.

### Deployment & Packaging (DEPL)

- [x] **DEPL-01**: Windows MSI packages (machine and user) are automatically generated and signed in CI.
- [x] **DEPL-02**: CLI provides a unified support status report on Windows via `nono setup --check-only`.

## v2.0 Requirements (Shipped 2026-04-18)

Gap closure milestone. Closes the 7 feature gaps between Windows and Unix platforms.

### Wrap (WRAP)

- [x] **WRAP-01**: User can run `nono wrap <cmd>` on Windows using Direct strategy with Job Object + WFP enforcement; help text documents that the supervisor stays alive as Job Object owner (no exec-replace, unlike Unix).
  - **Outcome:** Validated. Phase 07 (code) + Phase 14 plan 14-02 (help-text correction) + Phase 13 UAT P07-HV-1 pass, P07-HV-2 pass.

### Session Commands (SESS)

- [x] **SESS-01**: User can view logs for a Windows session using `nono logs <session>`.
- [x] **SESS-02**: User can inspect a Windows session record in JSON format using `nono inspect <session>`.
- [x] **SESS-03**: User can prune stale Windows session records using `nono prune`.
  - **Outcome:** Validated at the unit/integration level. Live round-trip UAT (P07-HV-3) waived as v2.0-known-issue — prereq on a live detached session which is blocked by Bug #3 residual. Code paths are exercised by `session_commands*` tests.

### Interactive Shell (SHELL)

- [x] **SHELL-01**: User can run `nono shell` to launch an interactive PowerShell or cmd.exe session inside a Job Object + WFP sandbox via ConPTY on Windows 10 build 17763+; terminal resize and Ctrl-C forwarding work correctly; no silent fallback to non-PTY path on unsupported builds.
  - **Outcome:** Validated. Phase 08 UAT-driven verification. (Planning-trail note: Phase 08 intentionally bypassed PLAN.md/SUMMARY.md/VERIFICATION.md in favor of UAT-only flow; documented in v2.0-MILESTONE-AUDIT.md `tech_debt`.)

### Network Policy (PORT, PROXY)

- [x] **PORT-01**: User can allow specific ports for outbound TCP connections on Windows using `--allow-port`; bind and connect allowlists operate independently; WFP permit filters have higher weight than the block-all filter.
  - **Outcome:** Validated. Phase 09 + P09-HV-2 integration test pass.
- [x] **PROXY-01**: User can route sandboxed agent traffic through a local proxy via `--network-profile`/`--credential`/`--upstream-proxy` with `HTTPS_PROXY` credential injection; a WFP loopback permit filter ensures the proxy port is reachable; all other outbound traffic is blocked.
  - **Outcome:** Validated at the code + unit/integration test level. Live end-to-end UAT (P09-HV-1) waived as `no-test-fixture` — no built-in network-profile-with-credentials ships out of the box; users with a configured network profile can verify via the corrected runbook.
  - **Correction:** The originally-documented flag `--proxy-only` does not exist. Real plumbing uses `--network-profile` + `--credential` + `--upstream-proxy`. Runbook fixed in Phase 14 plan 14-03 Task 1.

### Path Discovery (LEARN)

- [x] **LEARN-01**: User can run `nono learn <cmd>` on Windows to capture file and network access patterns via ETW; output format matches Unix learn format so existing profile tooling works unchanged; running without admin privilege produces a clear error rather than silent empty output.
  - **Outcome:** Validated. Phase 10 (ferrisetw + Kernel-Network provider + file event capture + Windows integration test).

### Runtime Expansion — Stretch (TRUST)

- [x] **TRUST-01** *(stretch)*: A sandboxed child process can request additional capabilities from the supervisor at runtime via named pipe; the supervisor prompts the user for approval before granting; requests without a valid session token are denied immediately with a constant-time comparison.
  - **Outcome:** Validated at the protocol + automated-test level. Live supervised cross-console UAT (P11-HV-1) and token-leak audit under real tracing subscriber (P11-HV-3) waived as v2.0-known-issue — the supervised detached path depends on Bug #3 resolution in Phase 15. Phase 11 automated checks (capability broker unit tests, SACL pipe roundtrip, approval-denied sequence, token redaction tests) are green. P11-HV-2 (cross-boundary LI pipe test) waived per pre-existing rationale — outside cargo-test scope.
