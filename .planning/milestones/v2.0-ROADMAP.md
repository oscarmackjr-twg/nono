This archive and the `v2.0` git tag are the authoritative names for the milestone closed on 2026-04-18.

## Overview

Close the 7 remaining feature gaps between Windows and Unix platforms so everyday CLI usage, network policy, and developer tooling reach cross-platform parity. Delivered against the 9 v2.0 requirements (WRAP, SESS, SHELL, PORT, PROXY, LEARN, TRUST stretch).

**Core value:** Windows security must be as structurally impossible and feature-complete as Unix platforms, ensuring the dangerous bits are kernel-enforced without compromising the supervisor-led security model.

## Milestone Summary

### Key Decisions

- **Supervisor-Broker Pattern** (2026-04-04) — The only way to manage elevated tasks like WFP while maintaining user-level CLI.
- **WFP as Primary Network Backend** (2026-04-04) — Moved away from temporary firewall rules for true kernel-level enforcement. Validated in Phase 6 (SID wiring end-to-end).
- **Named Job Objects** (2026-04-04) — For agent lifecycle management and atomic stop/list.
- **SID-Based Filtering prioritized over AppID** (2026-04-04) — To ensure child processes inherit network restrictions. Caveat: detached path regressed to AppID-dependency as a v2.0-known-issue (see Phase 14 14-01 SUMMARY).
- **Double-Launch Strategy** (2026-04-04) — `DETACHED_PROCESS` to decouple supervisor from parent terminal.
- **Restricted Tokens** (2026-04-04) — Apply session-unique SID to the process tree. `WRITE_RESTRICTED` fix for `STATUS_ACCESS_DENIED` (commit `e094994`, Phase 12). The `STATUS_DLL_INIT_FAILED (0xC0000142)` residual on detached console grandchildren is the v2.0-known-issue.
- **RFC 3161 Timestamping** (2026-04-05) — Upgraded from legacy `/t` to `/tr + /td sha256` in signing workflow.
- **WFP Startup Orphan Sweep** (2026-04-05) — Enumerates `NONO_SUBLAYER_GUID` filters and removes stale ones at startup.
- **Machine MSI Owns EventLog Registration** (2026-04-05) — `SYSTEM\CurrentControlSet\Services\EventLog\Application\nono-wfp-service`.
- **MSRV 1.77** (2026-04-05) — Bumped from 1.74 to align with windows-sys 0.59.
- **WaitNamedPipeW Readiness Probe** (2026-04-05) — `run_detached_launch()` uses `WaitNamedPipeW(50ms)` per iteration on Windows.
- **Ship v2.0 with the detached-console-grandchild bug as a documented known issue** (2026-04-18) — Explicit user decision after Phase 14 plan 14-01's three fix attempts all failed the smoke gate. Non-detached mode is fully functional; GUI apps in detached mode work. Carry-forward to Phase 15.

### Issues Resolved

- **Bug #1 (handle UAF in `spawn_windows_child`):** fixed 2026-04-17, commit `eb4730c` (quick task `260417-wla`). Token handle use-after-close that was blocking both Bug #2 and Bug #3 from being reachable.
- **Bug #2 (`0xC0000022 STATUS_ACCESS_DENIED` on all sandboxed children):** fixed 2026-04-17, commit `e094994`. `CreateRestrictedToken` was called with `Flags=0`; the restricting-SID access-check gate blocked every access. Fix: pass `WRITE_RESTRICTED`, narrowing the gate to write operations only.
- **Phase 07 P07-HV-2 (setup help text self-contradiction):** fixed by Phase 14 plan 14-02, commit `8e200f8`. Replaced stale `"remain intentionally unavailable"` line with canonical `'nono wrap' is available on Windows with Job Object + WFP enforcement (no exec-replace, unlike Unix)` sentence.
- **Phase 13 runbook defects (P05-HV-1 `--detach` typo, P09-HV-1 `--proxy-only` nonexistent flag):** fixed by Phase 14 plan 14-03 Task 1, commit `647e0a5`. Real flags are `--detached` and `--network-profile`/`--credential`/`--upstream-proxy`.

### Issues Deferred (v2.0-known-issue carry-forward to Phase 15)

- **Bug #3 residual (`0xC0000142 STATUS_DLL_INIT_FAILED` on detached console grandchildren):** documented in `.planning/debug/windows-supervised-exec-cascade.md`. Three fix directions tried in Phase 14 plan 14-01 — Direction 1 (`AdjustTokenGroups`) infeasible pre-commit; Direction 3 (`AllocConsole`) and Direction 2 (null token) both failed the smoke gate. Debug matrix shows the only working config is `null token + no PTY`, which requires PTY + detached-supervisor architecture work. Deferred to Phase 15 per explicit user shipping decision.
- **Affected Phase 13 UAT items carried forward:** P05-HV-1, P07-HV-3, P11-HV-1, P11-HV-3. All waived as `v2.0-known-issue` in `13-UAT.md`. Upstream VERIFICATION.md files (05, 07, 11) carry inline addendums flagging the carry-forward.
- **P09-HV-1 live end-to-end:** waived as `no-test-fixture`. The corrected runbook is verified on an admin Windows host with `nono-wfp-service` running, but no built-in network-profile-with-credentials ships out of the box, so the final env-injection assertion cannot run in this environment. Users with a configured network profile can verify via the corrected runbook.
