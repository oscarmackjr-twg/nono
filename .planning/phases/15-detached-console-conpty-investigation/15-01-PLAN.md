## Rejected Alternatives (cite before planning any new direction)

The following were attempted in Phase 14 and MUST NOT be re-proposed as primary paths:

1. **Direction 1 — Session SID as TokenGroups via AdjustTokenGroups (14-01 pre-commit):** Infeasible. `AdjustTokenGroups` cannot add new SIDs to a token; `SetTokenInformation(TokenGroups)` requires `SE_TCB_NAME` (SYSTEM only). No user-mode API can inject a synthetic SID as a token group. Fully documented in `14-01-SUMMARY.md § Direction 1 abandoned` and `14-01-PLAN.md <direction_decision>`.

2. **Direction 3 — AllocConsole in detached supervisor + WRITE_RESTRICTED token (commit b06aebe, reverted 1980df5):** Smoke-gate FAILED rows 1 and 2 with identical `0xC0000142`. AllocConsole alone does not unblock DLL loader init. Evidence: `14-01-SUMMARY.md § Smoke-gate results, Pass 1`.

3. **Direction 2 — Null token in detached mode with PTY still live (commit 005f6bc, reverted bd55893):** Smoke-gate FAILED rows 1 and 2 with identical `0xC0000142`. As the debug-doc matrix predicts, `null token + Some PTY → 0xC0000142`. The PTY must ALSO be disabled for the working configuration. Evidence: `14-01-SUMMARY.md § Smoke-gate results, Pass 2` and debug-doc matrix.
