**File 1: `crates/nono-cli/src/execution_runtime.rs`**

In the `execute_sandboxed` function, replace the Direct strategy match arm (lines 273-283) with platform-conditional logic. The current code is:

```rust
exec_strategy::ExecStrategy::Direct => {
    #[cfg(target_os = "windows")]
    {
        exec_strategy::execute_direct(&config, Some(flags.session.session_id.as_str()))?;
    }
    #[cfg(not(target_os = "windows"))]
    {
        exec_strategy::execute_direct(&config)?;
    }
    unreachable!("execute_direct only returns on error");
}
```

Replace with:

```rust
exec_strategy::ExecStrategy::Direct => {
    #[cfg(target_os = "windows")]
    {
        let exit_code = exec_strategy::execute_direct(&config, None)?;
        cleanup_capability_state_file(&cap_file_path);
        std::process::exit(exit_code);
    }
    #[cfg(not(target_os = "windows"))]
    {
        exec_strategy::execute_direct(&config)?;
    }
    #[allow(unreachable_code)]
    {
        unreachable!("execute_direct only returns on error");
    }
}
```

Key changes (per WRAP-01):
1. Capture the `Ok(i32)` return value from `execute_direct` into `exit_code`.
2. Pass `None` instead of `Some(flags.session.session_id.as_str())` -- wrap does not need a named Job Object (anonymous is correct since wrap has no detach/ps integration and `SessionLaunchOptions::default()` gives empty string which would produce a malformed Job Object name `Local\nono-session-`). Safety of `create_process_containment(None)` is pre-verified in `<verified_facts>`.
3. Call `cleanup_capability_state_file(&cap_file_path)` to clean up the capability state file before exiting (matches the Supervised strategy cleanup pattern at line 298).
4. Call `std::process::exit(exit_code)` to exit with the child's exit code instead of falling through to `unreachable!()`.
5. Add `#[allow(unreachable_code)]` around the `unreachable!()` to suppress compiler warnings -- on Windows `std::process::exit` diverges so the macro is unreachable; on Unix `execute_direct` never returns on success so the `?` propagates errors.
