<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User CLI arg `--allow-gpu` → sandbox backend | Flag opens privileged paths (NVIDIA device nodes on Linux, GPU frameworks on macOS). If the flag were default-on, it would broaden sandbox exposure silently. |
| `/dev/nvidia-uvm-tools` → kernel GPU privilege surface | This device has a known privilege-escalation history (search CVE DB for `nvidia-uvm` + year). Granting access broadens attack surface — accepted risk per upstream, gated on explicit opt-in. |
| Untrusted JWT from CI pipeline → GitLab ID token validator | GitLab issuer URL must be validated against a pinned configuration to prevent a malicious token from an unexpected issuer from passing signing validation. |
| Warning emission on Windows → operator awareness | The "--allow-gpu is not enforced on Windows" warning must surface clearly so operators do not falsely assume GPU passthrough is sandboxed. |
