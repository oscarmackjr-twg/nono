<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Profile config file → profile loader | Untrusted profile YAML/JSON enters the loader and its `extends` chain is traversed. A malicious or misconfigured profile whose `extends` cycles currently DoS's the process via stack exhaustion. |
| .claude.json symlink → claude-code config read path | An attacker who can plant a symlink chain in the claude-code config dir could redirect token-refresh reads to a file outside the intended config root. Canonicalization + `Path::starts_with` root-containment validation prevents this. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-02-01 | Denial of Service | Profile `extends` loader | mitigate | Cycle-detection guard ported from upstream c1bc439. On direct self-reference or indirect cycle, loader returns `NonoError::InvalidConfig` in bounded time instead of stack-overflowing. Unit-tested: direct self-ref, indirect two-hop cycle, linear-chain regression. |
| T-20-02-02 | Tampering | Profile loader accepts overly-deep legitimate chain | accept | Depth-bounded implementations (if upstream's guard is depth-based rather than visited-set-based) cap extends depth at upstream's constant. A legitimate chain exceeding the cap is rejected — acceptable vs the DoS risk; linear-chain regression test ensures reasonable chains still resolve. |
| T-20-02-03 | Tampering | Attacker-planted symlink on claude-code config path redirects token refresh | mitigate | Post-port, `.claude.json` symlink target is canonicalized via `std::fs::canonicalize` and validated to stay inside the claude-code config root via `Path::starts_with` (component comparison, NOT string starts_with — CLAUDE.md § Common Footgun #1). Fail-closed with `NonoError::InvalidConfig` on escape. Unit-tested with path-traversal hostile input. |
| T-20-02-04 | Denial of Service | Symlink creation fails on unprivileged Windows hosts (no Developer Mode, no SeCreateSymbolicLinkPrivilege) | mitigate | Windows symlink path catches `io::Error`, emits `tracing::warn!`, and returns `Ok(())` (fail-open for install; runtime behavior unchanged from the pre-port state — claude-code refresh simply does not happen via symlink on that host, same as today). Not a regression vs current fork behavior. |
| T-20-02-05 | Repudiation | Commit provenance for upstream ports | mitigate | Both commits carry `Upstream-commit:` + `Upstream-tag: v0.37.1` + `Upstream-author:` + DCO `Signed-off-by: Oscar Mack <oscar.mack.jr@gmail.com>` trailers. Manual-port variants explicitly note "Manual port of upstream <hash>" in the body so the audit trail is unambiguous. |
| T-20-02-06 | Elevation of Privilege | Fork path routes around upstream's intended guard | accept | Tests (`test_profile_extends_*`, claude.json path-traversal rejection) cover the guarded paths. Any future code path that bypasses them would be caught by the tests as a regression. |
</threat_model>
