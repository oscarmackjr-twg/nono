<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted credential-URI string (config file, env var, CLI arg) → keystore parser | A malicious or malformed credential URI enters the fork's credential-URI parser. If the parser reads the filesystem based on unsanitized URI components, an attacker could redirect credential reads to `/etc/shadow` (Unix) or `C:\Windows\System32\config\*` (Windows) via path traversal. |
| Parent process env → sandboxed child process env | Secrets or host-identifying strings in the parent env could leak to the sandboxed child if the filter admits them. Fail-closed behavior on malformed filter patterns prevents an attacker from bypassing the filter by crafting a pattern that "looks like" it filters but actually doesn't. |
| Deprecated command list → CLI dispatch path | The `command_blocking_deprecation.rs` backport emits warnings. A malicious backport could flip a previously-blocked command's enforcement disposition under the guise of "deprecation" — explicitly non-goal of this plan (warnings only). |
