<objective>
Implement RESL-03 — the `--timeout <DURATION>` wall-clock supervisor timer — and add the observability surface for all four resource limits in `nono inspect`.

Purpose: Close the final RESL acceptance criterion in REQUIREMENTS.md. The timeout mechanism is deliberately NOT a `JOB_OBJECT_LIMIT_JOB_TIME` kernel bit (that limit tracks CPU time, not wall-clock; see CONTEXT.md § Windows Enforcement line 41). Instead, the supervisor event loop checks an `Instant` deadline each iteration and calls `TerminateJobObject` when it expires. The `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE` flag preserved by Plan 16-01 remains the safety net: if the supervisor itself crashes before the deadline, the Job Object still dies cleanly.

This plan also adds a `limits: Option<ResourceLimitsRecord>` field to `SessionRecord` so `nono inspect <id>` can surface the active caps to operators — closing the observability gap called out in CONTEXT.md § Specific Implementation Pointers.

Output: Working `nono run --timeout 5s -- ping -t 127.0.0.1` that terminates after ~5 seconds with a recognizable exit code; `nono inspect <id>` showing a formatted `Limits:` block; all four RESL requirements shipped; `make ci` green.

Depends on Plan 16-01 for the `ResourceLimits` struct, the CLI flag plumbing, and the CPU/memory/max-process enforcement.

<serialization_compat>
**Backward compatibility matters for `SessionRecord`:**

Sessions written before this plan have no `limits` field on disk. The new field MUST use:
```rust
#[serde(default, skip_serializing_if = "Option::is_none")]
pub limits: Option<ResourceLimitsRecord>,
```

- `#[serde(default)]` → old files deserialize cleanly with `limits=None`.
- `skip_serializing_if = "Option::is_none"` → sessions with no limits set do not add noise to the on-disk JSON.
- `ResourceLimitsRecord` itself should have all four `Option<T>` fields with the same pair of attributes, so a session with only `--cpu-percent` set serializes as `"limits": { "cpu_percent": 25 }` (not the other three keys with `null`).

**Unit-test the backward-compat explicitly:** deserialize a JSON blob without the `limits` field and assert it loads cleanly. This catches accidental removal of `#[serde(default)]`.
</serialization_compat>

<timeout_accuracy>
**Expected timeout accuracy:** within ±200ms of the requested duration, limited by:
- The event loop's `wait_for_exit(100)` poll interval (100ms quanta).
- OS scheduler jitter on `Instant::now()` measurement.

This is acceptable for the acceptance criterion "kill after 5s" — users are not relying on sub-second precision for a sandbox wall-clock limit. Document this in the `///` doc comment on the deadline-check logic.

**Edge cases the implementation must handle:**
1. **Very short timeouts (`--timeout 1s`):** the first `wait_for_exit(100)` might complete before the first deadline check. The loop structure (check deadline BEFORE `wait_for_exit`) means the first iteration might wait 100ms past the deadline. Acceptable.
2. **Deadline math overflow:** `Instant::now() + Duration::from_secs(u64::MAX)` can panic on some platforms. Use `Instant::now().checked_add(duration)` and on `None` → `NonoError::SandboxInit("--timeout value exceeds platform Instant range")` at supervisor init time (before the spawn commits). This is a platform-specific edge case; the clap parser already prevents duration=0 but does not prevent u64::MAX seconds.
3. **`TerminateJobObject` itself fails:** if the FFI call returns 0, log the GetLastError at tracing::error and return the existing `CommandExecution` error — the grandchild will still die when the supervisor drops its job handle (kill-on-close safety net). Do NOT retry in a loop.
</timeout_accuracy>

    **Step 1 — Add `terminate_job_object` helper to `launch.rs`.**

    Alongside the existing `terminate_suspended_process` helper (around line 113):

    ```rust
    /// Status code embedded in the Job Object's exit code when the supervisor
    /// terminates the tree for a `--timeout` wall-clock expiry. Equal to
    /// `STATUS_TIMEOUT` / `WAIT_TIMEOUT` (0x00000102 = 258 decimal). Callers of
    /// `nono run --timeout ...` see this as the supervisor's exit code on expiry.
    pub(super) const STATUS_TIMEOUT_EXIT_CODE: u32 = 0x0000_0102;

    /// Terminate every process in the given Job Object with the supplied exit
    /// code. Used by the supervisor to honor `--timeout` (RESL-03) and by any
    /// future supervisor-initiated kill paths.
    ///
    /// Returns `Err(NonoError::CommandExecution(...))` when the FFI call fails;
    /// the caller should log the error. The Job Object's own
    /// `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE` flag remains the safety net if this
    /// FFI call misfires — see CONTEXT.md § Failure Modes.
    pub(super) fn terminate_job_object(job: HANDLE, exit_code: u32) -> Result<()> {
        let ok = unsafe {
            // SAFETY: `job` is a live Job Object handle borrowed from
            // ProcessContainment; TerminateJobObject accepts any HANDLE the
            // caller has JOB_OBJECT_TERMINATE access to.
            TerminateJobObject(job, exit_code)
        };
        if ok == 0 {
            return Err(NonoError::CommandExecution(std::io::Error::other(format!(
                "TerminateJobObject failed (exit_code={}, GetLastError={})",
                exit_code,
                unsafe { GetLastError() }
            ))));
        }
        Ok(())
    }
    ```

    ```rust
    /// Compute the absolute deadline Instant for a supervisor-side wall-clock
    /// timeout. Returns None when no timeout is requested; returns Err when
    /// the requested duration would overflow `Instant` arithmetic on this
    /// platform (a theoretical edge for u64::MAX seconds that the clap parser
    /// does not prevent).
    pub(super) fn compute_deadline(
        timeout: Option<std::time::Duration>,
        now: std::time::Instant,
    ) -> Result<Option<std::time::Instant>> {
        match timeout {
            None => Ok(None),
            Some(d) => now
                .checked_add(d)
                .map(Some)
                .ok_or_else(|| {
                    NonoError::SandboxInit(format!(
                        "--timeout value exceeds platform Instant range: {d:?}"
                    ))
                }),
        }
    }
    ```

    **Step 4 — Add the deadline check to `run_child_event_loop`.**

    Replace the existing loop body with the extended version. The check goes BEFORE `wait_for_exit` but AFTER `terminate_requested`:

    ```rust
    loop {
        self.drain_capability_audit_entries();
        if self.terminate_requested.load(Ordering::SeqCst) {
            tracing::info!(
                "Windows supervisor received termination request, stopping child..."
            );
            child.terminate()?;
            return Ok(-1);
        }

        // Wall-clock deadline (RESL-03 --timeout).
        if let Some(deadline) = self.timeout_deadline {
            if std::time::Instant::now() >= deadline {
                tracing::info!(
                    "Windows supervisor --timeout expired (session: {}, elapsed_ms: {}), calling TerminateJobObject...",
                    self.session_id,
                    self.started_at.elapsed().as_millis()
                );
                // Best-effort kill; if this fails the tree still dies when
                // the job handle closes (KILL_ON_JOB_CLOSE safety net).
                if let Err(err) = crate::exec_strategy_windows::launch::terminate_job_object(
                    self.containment_job,
                    crate::exec_strategy_windows::launch::STATUS_TIMEOUT_EXIT_CODE,
                ) {
                    tracing::error!(
                        "TerminateJobObject on --timeout expiry failed (will rely on KILL_ON_JOB_CLOSE safety net): {err}"
                    );
                }
                self.state = WindowsSupervisorLifecycleState::ShuttingDown;
                self.shutdown();
                self.state = WindowsSupervisorLifecycleState::Completed;
                return Ok(
                    crate::exec_strategy_windows::launch::STATUS_TIMEOUT_EXIT_CODE as i32,
                );
            }
        }

        if let Some(exit_code) = child.wait_for_exit(100)? {
            self.state = WindowsSupervisorLifecycleState::ShuttingDown;
            self.shutdown();
            self.state = WindowsSupervisorLifecycleState::Completed;
            return Ok(exit_code);
        }
    }
    ```

    ```rust
    /// Serialized view of the resource limits that were active when a session
    /// was launched. Lives on `SessionRecord.limits`. Each field is
    /// independently optional; unset fields are omitted from the JSON.
    ///
    /// Stable JSON field names:
    /// - `cpu_percent` (u16, 1..=100)
    /// - `memory_bytes` (u64)
    /// - `timeout_seconds` (u64, duration expressed in whole seconds)
    /// - `max_processes` (u32)
    ///
    /// See `nono inspect` for human-readable rendering.
    #[derive(Debug, Clone, Default, Serialize, Deserialize, PartialEq, Eq)]
    pub struct ResourceLimitsRecord {
        #[serde(default, skip_serializing_if = "Option::is_none")]
        pub cpu_percent: Option<u16>,
        #[serde(default, skip_serializing_if = "Option::is_none")]
        pub memory_bytes: Option<u64>,
        #[serde(default, skip_serializing_if = "Option::is_none")]
        pub timeout_seconds: Option<u64>,
        #[serde(default, skip_serializing_if = "Option::is_none")]
        pub max_processes: Option<u32>,
    }

    impl ResourceLimitsRecord {
        /// Build a record from the in-memory `ResourceLimits` struct
        /// (`launch_runtime::ResourceLimits`). Returns `None` when every field
        /// is `None`, so a session that did not set any limits does not write
        /// an empty `"limits": {}` object to disk.
        pub(crate) fn from_resource_limits(
            limits: &crate::launch_runtime::ResourceLimits,
        ) -> Option<Self> {
            if limits.is_empty() {
                return None;
            }
            Some(Self {
                cpu_percent: limits.cpu_percent,
                memory_bytes: limits.memory_bytes,
                timeout_seconds: limits.timeout.map(|d| d.as_secs()),
                max_processes: limits.max_processes,
            })
        }

        pub fn is_empty(&self) -> bool {
            self.cpu_percent.is_none()
                && self.memory_bytes.is_none()
                && self.timeout_seconds.is_none()
                && self.max_processes.is_none()
        }
    }
    ```

    ```rust
    /// Human-readable bytes: picks the largest unit that yields an integer representation.
    /// `512 MiB` (binary units) — matches the input parser's 1024-based multipliers from Plan 16-01.
    fn format_bytes_human(bytes: u64) -> String {
        const K: u64 = 1024;
        const M: u64 = K * 1024;
        const G: u64 = M * 1024;
        const T: u64 = G * 1024;
        if bytes >= T && bytes % T == 0 {
            format!("{} TiB", bytes / T)
        } else if bytes >= G && bytes % G == 0 {
            format!("{} GiB", bytes / G)
        } else if bytes >= M && bytes % M == 0 {
            format!("{} MiB", bytes / M)
        } else if bytes >= K && bytes % K == 0 {
            format!("{} KiB", bytes / K)
        } else {
            format!("{} bytes", bytes)
        }
    }

    /// Human-readable duration: "5 minutes" / "1 hour" / "45 seconds".
    /// Not a general-purpose formatter — tuned for the durations our CLI parser accepts.
    fn format_duration_human(d: std::time::Duration) -> String {
        let secs = d.as_secs();
        if secs >= 86_400 && secs % 86_400 == 0 {
            let n = secs / 86_400;
            format!("{n} {}", if n == 1 { "day" } else { "days" })
        } else if secs >= 3600 && secs % 3600 == 0 {
            let n = secs / 3600;
            format!("{n} {}", if n == 1 { "hour" } else { "hours" })
        } else if secs >= 60 && secs % 60 == 0 {
            let n = secs / 60;
            format!("{n} {}", if n == 1 { "minute" } else { "minutes" })
        } else {
            format!("{secs} {}", if secs == 1 { "second" } else { "seconds" })
        }
    }
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| --timeout flag value → supervisor deadline arithmetic | Untrusted duration could in theory overflow Instant arithmetic on some platforms. |
| supervisor event loop → TerminateJobObject FFI | The FFI call is trusted, but failure must not leave the supervisor in an inconsistent state. |
| SessionRecord JSON file on disk → next nono invocation | On-disk file is written by the supervisor and read by `nono inspect` / `nono ps`. Format is an append-only contract; renames break nono across versions. |
| Borrowed containment.job HANDLE → WindowsSupervisorRuntime | Runtime holds a borrowed copy; must not CloseHandle it. Double-close is UB. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02-01 | Denial of Service | `--timeout 5s` on a busy/scheduled host could fire slightly after 5s — user expects ≤5s, gets ≤5.2s. | accept | Documented in `compute_deadline` doc comment and in the 16-02-SUMMARY "Known Caveats" section. Not a security issue; it is a precision caveat. |
| T-16-02-02 | Elevation of Privilege | If the event loop skips the deadline check in some iteration, the child overruns the cap. | mitigate | The check is the FIRST iteration step after `terminate_requested`, before `wait_for_exit`. | T-16-02-03 | Tampering | `--timeout u64::MAX` overflows `Instant::now() + Duration::from_secs(u64::MAX)` on some platforms, causing a panic. | mitigate | `compute_deadline` uses `checked_add` and returns `NonoError::SandboxInit` on overflow; the supervisor aborts before spawning. Unit-tested. |
| T-16-02-04 | Denial of Service | `TerminateJobObject` failure leaves the supervisor returning a timeout exit code while the child is still alive. | mitigate | `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE` (preserved by Plan 16-01) is the safety net — when the runtime drops its borrowed handle and `ProcessContainment::Drop` closes the owning handle, the kernel kills the tree.
