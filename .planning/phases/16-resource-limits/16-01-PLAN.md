Splitting keeps each plan under ~30% context and gives Plan 16-02 a clean dependency on 16-01's CLI + struct plumbing.

- Suffix multipliers: K=1024, M=1024*1024, G=1024*1024*1024, T=1024^4.
- Reject: `0`, negative, empty, `NaN`, overflow (return `Err("memory value overflowed u64")`).
- Reject stray whitespace inside the value.
- Live in `crates/nono-cli/src/cli.rs` as `fn parse_byte_size(s: &str) -> Result<u64, String>`; wired via clap `value_parser = parse_byte_size`.

**Duration parser (`parse_duration(&str) -> Result<std::time::Duration, String>`):**

- Reject: `0`, negative, empty, malformed, overflow past u64::MAX seconds.
- Live alongside `parse_byte_size` in `cli.rs`; wired via clap `value_parser = parse_duration`.

    ```rust
    /// Parse a byte-size value of the form `512M` / `1G` / `256K` / raw bytes `268435456`.
    /// Suffix is case-insensitive; multipliers are 1024-based (K=1024, M=1024^2, G=1024^3, T=1024^4).
    /// Rejects `0`, negative, empty, unrecognized suffixes, and values that overflow u64.
    pub(crate) fn parse_byte_size(s: &str) -> std::result::Result<u64, String> {
        let s = s.trim();
        if s.is_empty() {
            return Err("memory size cannot be empty".to_string());
        }
        // Split the numeric head from the single-char suffix, if any.
        let (num_str, multiplier): (&str, u64) = match s.chars().last() {
            Some(c) if c.is_ascii_alphabetic() => {
                let mult: u64 = match c.to_ascii_uppercase() {
                    'K' => 1024,
                    'M' => 1024 * 1024,
                    'G' => 1024 * 1024 * 1024,
                    'T' => 1024u64.pow(4),
                    other => return Err(format!("unrecognized memory suffix '{other}'; expected K/M/G/T")),
                };
                (&s[..s.len() - 1], mult)
            }
            _ => (s, 1),
        };
        let n: u64 = num_str.parse().map_err(|_| format!("invalid memory value '{s}'"))?;
        if n == 0 {
            return Err("memory value must be > 0".to_string());
        }
        n.checked_mul(multiplier)
            .ok_or_else(|| format!("memory value '{s}' overflows u64"))
    }

    /// Parse a duration of the form `30s` / `5m` / `1h` / `1d` / raw seconds `300`.
    /// Rejects `0`, negative, empty, unrecognized suffixes, and overflow past u64 seconds.
    pub(crate) fn parse_duration(s: &str) -> std::result::Result<std::time::Duration, String> {
        let s = s.trim();
        if s.is_empty() {
            return Err("timeout cannot be empty".to_string());
        }
        let (num_str, multiplier): (&str, u64) = match s.chars().last() {
            Some(c) if c.is_ascii_alphabetic() => {
                let mult: u64 = match c.to_ascii_lowercase() {
                    's' => 1,
                    'm' => 60,
                    'h' => 3600,
                    'd' => 86_400,
                    other => return Err(format!("unrecognized duration suffix '{other}'; expected s/m/h/d")),
                };
                (&s[..s.len() - 1], mult)
            }
            _ => (s, 1),
        };
        let n: u64 = num_str.parse().map_err(|_| format!("invalid duration '{s}'"))?;
        if n == 0 {
            return Err("timeout must be > 0".to_string());
        }
        let secs = n.checked_mul(multiplier).ok_or_else(|| format!("duration '{s}' overflows u64 seconds"))?;
        Ok(std::time::Duration::from_secs(secs))
    }
    ```

    Pseudocode sketch:
    ```rust
    pub(super) fn apply_resource_limits(
        containment: &ProcessContainment,
        limits: &ResourceLimits,
    ) -> Result<()> {
        if limits.is_empty() {
            return Ok(());
        }

        // CPU rate control — distinct info class.
        if let Some(percent) = limits.cpu_percent {
            let mut info: JOBOBJECT_CPU_RATE_CONTROL_INFORMATION = unsafe {
                // SAFETY: plain Win32 FFI struct; zero-init is valid.
                std::mem::zeroed()
            };
            info.ControlFlags = JOB_OBJECT_CPU_RATE_CONTROL_ENABLE | JOB_OBJECT_CPU_RATE_CONTROL_HARD_CAP;
            // SAFETY: `Anonymous` is a C union; CpuRate is the active variant when HARD_CAP is set
            // (per Microsoft Docs: "use CpuRate unless WEIGHT_BASED is set"). WEIGHT_BASED is not in ControlFlags.
            unsafe { info.Anonymous.CpuRate = (percent as u32) * 100; }

            let ok = unsafe {
                // SAFETY: `info` is initialized; struct size matches the info class.
                SetInformationJobObject(
                    containment.job,
                    JobObjectCpuRateControlInformation,
                    &info as *const _ as *const _,
                    std::mem::size_of::<JOBOBJECT_CPU_RATE_CONTROL_INFORMATION>() as u32,
                )
            };
            if ok == 0 {
                return Err(NonoError::SandboxInit(format!(
                    "Failed to apply --cpu-percent={} to Windows Job Object (GetLastError={})",
                    percent,
                    unsafe { GetLastError() }
                )));
            }
        }

        // Memory + max-processes — share JOBOBJECT_EXTENDED_LIMIT_INFORMATION.
        if limits.memory_bytes.is_some() || limits.max_processes.is_some() {
            let mut info: JOBOBJECT_EXTENDED_LIMIT_INFORMATION = unsafe {
                // SAFETY: plain FFI struct; zero-init is valid baseline before the readback fills it.
                std::mem::zeroed()
            };
            let mut returned: u32 = 0;
            let ok = unsafe {
                // SAFETY: `info` sized exactly for the info class.
                QueryInformationJobObject(
                    containment.job,
                    JobObjectExtendedLimitInformation,
                    &mut info as *mut _ as *mut _,
                    std::mem::size_of::<JOBOBJECT_EXTENDED_LIMIT_INFORMATION>() as u32,
                    &mut returned,
                )
            };
            if ok == 0 {
                return Err(NonoError::SandboxInit(format!(
                    "Failed to read current Windows Job Object extended limit info (GetLastError={})",
                    unsafe { GetLastError() }
                )));
            }

            if let Some(mem) = limits.memory_bytes {
                info.BasicLimitInformation.LimitFlags |= JOB_OBJECT_LIMIT_JOB_MEMORY;
                info.JobMemoryLimit = mem as usize;
            }
            if let Some(procs) = limits.max_processes {
                info.BasicLimitInformation.LimitFlags |= JOB_OBJECT_LIMIT_ACTIVE_PROCESS;
                info.BasicLimitInformation.ActiveProcessLimit = procs;
            }

            let ok = unsafe {
                // SAFETY: `info` was initialized by the Query above and mutated in-place; size matches info class.
                SetInformationJobObject(
                    containment.job,
                    JobObjectExtendedLimitInformation,
                    &info as *const _ as *const _,
                    std::mem::size_of::<JOBOBJECT_EXTENDED_LIMIT_INFORMATION>() as u32,
                )
            };
            if ok == 0 {
                let which = match (limits.memory_bytes.is_some(), limits.max_processes.is_some()) {
                    (true, true) => "--memory + --max-processes",
                    (true, false) => "--memory",
                    (false, true) => "--max-processes",
                    (false, false) => "(none)",
                };
                return Err(NonoError::SandboxInit(format!(
                    "Failed to apply {} to Windows Job Object (GetLastError={})",
                    which,
                    unsafe { GetLastError() }
                )));
            }
        }

        Ok(())
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → nono CLI argument parser | Untrusted numeric / byte-size / duration strings reach the parsers. Must validate before touching the Job Object. |
| nono supervisor → Windows kernel (SetInformationJobObject) | The kernel trusts the caller to supply valid struct sizes + info class. Size mismatch or partial init is a UB hazard. |
| create_process_containment flags (KILL_ON_JOB_CLOSE, DIE_ON_UNHANDLED_EXCEPTION) → apply_resource_limits | If apply_resource_limits writes without reading-then-OR-ing, the baseline safety flags are lost and every session becomes a zombie-child risk on supervisor crash. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01-01 | Denial of Service | Malicious or buggy caller supplies `--memory 1T` / `--max-processes 65535` — legitimate, but could exhaust host resources if user intent is malformed. | accept | User is the same entity that runs the host. nono does not protect the host from its operator; fail-closed is about child bypass, not host DoS. |
| T-16-01-02 | Tampering | Byte-size parser overflow — `--memory 18446744073709551616` (u64::MAX + 1) could wrap to a small value and silently cap memory too low. | mitigate | `parse_byte_size` uses `checked_mul` on the suffix multiply and rejects overflow with an explicit error before clap accepts the arg. Unit-tested. |
| T-16-01-03 | Elevation of Privilege | `apply_resource_limits` clobbers `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE` by writing `JOBOBJECT_EXTENDED_LIMIT_INFORMATION` with only the new flag bit — supervisor crash would leave grandchild alive outside the Job Object's "die on close" guarantee. | mitigate | Read-modify-write pattern: Query the current struct first, OR in the new flags, then Set. | T-16-01-04 | Spoofing | Unix warning path is silent on `--silent` — a log-reader reviewing a silent Unix run may miss that `--cpu-percent` was accepted-but-unenforced. | accept | `--silent` is an explicit operator opt-in. | T-16-01-05 | Information Disclosure | GetLastError values embedded in the NonoError::SandboxInit message could leak OS-version fingerprints to a shared-terminal observer. | accept | nono error messages already embed GetLastError in existing paths (`create_process_containment`, `apply_process_handle_to_containment`).
