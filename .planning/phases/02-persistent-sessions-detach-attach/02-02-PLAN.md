<task type="auto">
  <name>Task 1: Implement Mandated Double-Launch Daemonization</name>
  <files>crates/nono-cli/src/execution_runtime.rs, crates/nono-cli/src/main.rs</files>
  <action>
    Implement the Double-Launch pattern as specified:
    1. Parent CLI detects the `--detach` flag.
    2. Re-launches its own binary with the hidden `--internal-supervisor` flag.
    3. The re-launch MUST use the Win32 flags: `DETACHED_PROCESS` and `CREATE_NEW_PROCESS_GROUP`.
    4. If the parent is in a Job Object that allows it, also use `CREATE_BREAKAWAY_FROM_JOB`.
    5. The second process (internal supervisor) spawns the actual agent child (in its own Job Object) and manages the IPC/PTY.
    6. Parent CLI exits immediately after successful re-launch.
  </action>
  <verify>
    <automated>cargo check -p nono-cli</automated>
  </verify>
  <done>Double-launch pattern correctly daemonizes the supervisor on Windows using DETACHED_PROCESS and CREATE_BREAKAWAY_FROM_JOB.</done>
</task>
