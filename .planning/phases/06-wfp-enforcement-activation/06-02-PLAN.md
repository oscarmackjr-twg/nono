**Part A: Extract `install_wfp_network_backend_with_runner` with BOTH probe and run closures**

Create a generic version of `install_wfp_network_backend` that accepts TWO closure parameters — one for the probe status check and one for the activation request — following the established `_with_runner` pattern but extending it to also inject the probe:

```rust
pub(super) fn install_wfp_network_backend_with_runner<P, R>(
    policy: &nono::WindowsNetworkPolicy,
    config: &ExecConfig<'_>,
    probe_config: &WfpProbeConfig,
    probe_fn: P,
    run_probe: R,
) -> Result<Option<NetworkEnforcementGuard>>
where
    P: Fn(&WfpProbeConfig) -> Result<WfpProbeStatus>,
    R: Fn(&WfpProbeConfig, &WfpRuntimeActivationRequest) -> Result<WfpRuntimeProbeOutput>,
```

Move the body of `install_wfp_network_backend` into this function:
- Replace the call to `probe_wfp_backend_status_with_config(probe_config)` (around line 1402) with `probe_fn(probe_config)`
- Replace the call to `run_wfp_runtime_probe_with_request(probe_config, &request)` (around line 1419) with `run_probe(probe_config, &request)`

Then update `install_wfp_network_backend` to delegate:
```rust
pub(super) fn install_wfp_network_backend(
    policy: &nono::WindowsNetworkPolicy,
    config: &ExecConfig<'_>,
    probe_config: &WfpProbeConfig,
) -> Result<Option<NetworkEnforcementGuard>> {
    install_wfp_network_backend_with_runner(
        policy,
        config,
        probe_config,
        probe_wfp_backend_status_with_config,
        run_wfp_runtime_probe_with_request,
    )
}
```

**WHY both closures:** The `probe_fn` closure is essential because `probe_wfp_backend_status_with_config` checks `config.backend_binary_path.exists()` even in the `windows_wfp_test_force_ready()` branch. On CI where `nono-wfp-service.exe` does not exist, the probe returns `BackendBinaryMissing` before the activation request is ever built, making the mock `run_probe` closure unreachable. Injecting the probe closure lets tests return `WfpProbeStatus::Ready` directly without any filesystem dependency.
