<objective>
Introduce age-based session retention for the Windows supervisor while keeping the existing Unix behavior compatible. Three deliverables land in this plan:

1. **Retention rule in code** (D-14): age-based, default 30 days, applied ONLY to `Status: Exited` sessions. Active (`Running` / `Paused`) sessions are NEVER pruned regardless of age. Fail-closed: if a session file's status field is unreadable or corrupt, the predicate returns `false` (do NOT prune).

2. **Auto-trigger in `nono ps`** (D-15): at the top of `run_ps`, compute stale-session count. If it exceeds the threshold (default 100, tune within 50-500 per CONTEXT.md discretion), invoke prune in a background thread and log `info: pruned N stale session files (>30 days, exited)` to stderr. Does NOT delay the `ps` table output. Structurally disabled inside sandboxes (NONO_CAP_FILE present).

3. **Extended `nono prune` CLI** (D-16): add `--older-than <duration>` using `parse_duration` (accepts `30d`/`5m`/raw seconds — a superset of the existing `--older-than <DAYS>`), `--all-exited` (ignore age, prune all exited), keep existing `--dry-run` and `--keep`. Accompanied by a unit test that validates the predicate (REQUIREMENTS CLEAN-04 acceptance #3).

5. **Docs** (D-18): `docs/session-retention.md` — rule, trigger, knobs, one-time cleanup note.

Purpose: Session files are a debt that accumulates silently; this plan bounds the backlog at 100 stale + whatever recent sessions exist, and gives operators both an automatic broom (`nono ps` trigger) and a manual one (`nono prune`).

    1. Add a pure predicate to `crates/nono-cli/src/session.rs` (place it near the other `pub fn` declarations, before `list_sessions`).

       **Boundary semantics (pinned by this plan):** the predicate uses an INCLUSIVE boundary — a session whose age equals `retention_secs` exactly IS prunable. Rationale: operators intuitively read "older than 30d" as "at or past the 30d mark"; making the boundary inclusive matches that intuition and removes an off-by-one class of surprise. CONTEXT.md D-14 does not specify the boundary, so this plan pins it here and documents it in the doc comment, the code (`>=`), and a dedicated boundary test.

       ```rust
       /// Returns true iff `record` is safe to prune under the retention rule.
       ///
       /// A record is prunable only if ALL of the following hold:
       /// - Its status is `SessionStatus::Exited` (never prune Running or Paused).
       /// - Its `started_epoch` is **at least** `retention_secs` before `now_epoch`
       ///   (inclusive boundary: a session exactly `retention_secs` old qualifies).
       ///
       /// `retention_secs = 0` combined with `Status: Exited` is the `--all-exited`
       /// escape hatch: every exited session qualifies (including just-exited ones).
       ///
       /// Fail-closed: if `now_epoch < record.started_epoch` (clock skew or corrupt
       /// record), returns `false` — do NOT prune.
       #[must_use]
       pub fn is_prunable(record: &SessionRecord, now_epoch: u64, retention_secs: u64) -> bool {
           if record.status != SessionStatus::Exited {
               return false;
           }
           // saturating_sub returns 0 if started_epoch > now_epoch (clock skew):
           //   - age == 0 with retention_secs > 0  → age < retention → not prunable (correct: fail closed)
           //   - age == 0 with retention_secs == 0 → age >= retention → prunable (--all-exited case)
           let age = now_epoch.saturating_sub(record.started_epoch);
           age >= retention_secs
       }
       ```

       Note the arithmetic uses `saturating_sub` per CLAUDE.md § "Arithmetic" — no risk of underflow on a malicious/corrupt `started_epoch` in the future.

       **Why the extra `age > 0 || retention_secs == 0` clause was dropped:** the simpler `age >= retention_secs` already handles both branches correctly. When `age == 0 && retention_secs == 0`, `0 >= 0` is true → prunable (correct: --all-exited includes just-exited). When `age == 0 && retention_secs > 0`, `0 >= retention_secs` is false → not prunable (correct: clock-skew fail-closed). Simpler expression, same behavior, easier to audit.

    1. **Extend `PruneArgs` in `cli.rs`** (around line 1993):

       Because the existing `--older-than` takes `u64 DAYS` and we want `--older-than <duration>` per D-16, introduce a REQUIRE-suffix adapter to preserve safety:

       ```rust
       /// Parse a duration for prune's `--older-than`, REQUIRING a suffix.
       ///
       /// Unlike `parse_duration` directly, this rejects raw integers because
       /// `--older-than 30` used to mean "30 days" (pre-CLEAN-04) but would
       /// suddenly mean "30 seconds" under parse_duration's rules. We refuse
       /// ambiguous input instead of silently changing behavior.
       fn parse_prune_duration(s: &str) -> std::result::Result<std::time::Duration, String> {
           // A suffix is any trailing non-digit char. parse_duration accepts both
           // raw ("30") and suffixed ("30d") forms; we gate on the former.
           if s.chars().all(|c| c.is_ascii_digit()) {
               return Err(format!(
                   "ambiguous duration '{s}' — please specify a suffix: \
                    {s}s (seconds), {s}m (minutes), {s}h (hours), {s}d (days)"
               ));
           }
           parse_duration(s)
       }

       #[derive(Parser, Debug)]
       pub struct PruneArgs {
           /// Show what would be removed without deleting
           #[arg(long)]
           pub dry_run: bool,

           /// Remove exited sessions older than this duration.
           /// Accepts 30s, 5m, 1h, 30d (no raw integers — specify a suffix).
           /// Default when unset: 30 days.
           #[arg(long, value_name = "DURATION", value_parser = parse_prune_duration)]
           pub older_than: Option<std::time::Duration>,

           /// Ignore age and prune every exited session (escape hatch).
           /// Mutually exclusive with --older-than.
           #[arg(long, conflicts_with = "older_than")]
           pub all_exited: bool,

           /// Keep only the N most recent sessions
           #[arg(long, value_name = "N")]
           pub keep: Option<usize>,
       }
       ```

       ```rust
           let now_epoch = match std::time::SystemTime::now().duration_since(std::time::UNIX_EPOCH) {

               Err(e) => {

               30 * 86_400 // default 30 days
           };

               .iter()
               .filter(|s| session::is_prunable(s, now_epoch, retention_secs))
               .collect();

           // list_sessions() is sorted newest-first.
           if let Some(keep) = args.keep {
               if to_remove.len() > keep {
                   to_remove = to_remove[keep..].to_vec();
               } else {
                   to_remove.clear();
               }
           }

           if to_remove.is_empty() {
               eprintln!("Nothing to prune.");
               return Ok(());
           }

           let dir = session::sessions_dir()?;

           for s in &to_remove {
               let session_file = dir.join(format!("{}.json", s.session_id));
               let events_file = dir.join(format!("{}.events.ndjson", s.session_id));

               if args.dry_run {
                   eprintln!("Would remove: {} (started {})", s.session_id, s.started);
               } else {
                   // Validate the target path is inside the canonical sessions dir
                   // (defense against symlink escape — threat T-19-04-02).
                   match session_file.canonicalize() {
                       Ok(resolved) if !resolved.starts_with(&dir) => {
                           debug!("Refusing to prune {}: outside sessions dir", session_file.display());
                           continue;
                       }
                       Err(_) => {
                           // File may not exist (already gone); fall through to remove_file which also errs-benignly.
                       }
                       _ => {}
                   }
                   // Reject symlinks: use symlink_metadata to check file type without following.
                   match std::fs::symlink_metadata(&session_file) {
                       Ok(md) if md.file_type().is_symlink() => {
                           debug!("Refusing to prune {}: is a symlink", session_file.display());
                           continue;
                       }
                       _ => {}
                   }
                   if let Err(e) = std::fs::remove_file(&session_file) {
                       debug!("Failed to remove session file {}: {}", session_file.display(), e);
                   }
                   if events_file.exists() {
                       if let Err(e) = std::fs::remove_file(&events_file) {
                           debug!("Failed to remove events file {}: {}", events_file.display(), e);
                       }
                   }
                   eprintln!("Removed: {} (started {})", s.session_id, s.started);
               }
           }

           eprintln!(
               "\n{} {} session(s).",
               if args.dry_run { "Would prune" } else { "Pruned" },
               to_remove.len()
           );

           Ok(())
       }
       ```

       ```rust
           std::thread::spawn(move || {

                   Ok(d) => d,

                       return;

               }

           });
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Disk ↔ nono supervisor (`~/.nono/sessions/`) | Session JSON files are untrusted input: a malicious agent running under nono could craft files with symlinks pointing outside the sessions dir, attempting to coax `nono prune` into deleting arbitrary files. |
| Clock ↔ retention predicate | `started_epoch` in a session file can be set to any `u64`; a corrupt or tampered value (e.g. `u64::MAX` or a future timestamp) could make an active session appear ancient or an exited session appear young. |
| Process ↔ filesystem (prune) | During prune, between the "is this session Exited?" check and the `remove_file` call, the state can change (TOCTOU). |
| Sandboxed agent ↔ host supervisor's session dir | An agent running inside nono still has access to env, argv, and (if it can reach a `nono` binary) the ability to run `nono ps`. `nono ps` must not be a vehicle for deleting the host's session files. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-04-01 | Tampering | `SessionRecord.status` field in an on-disk JSON | mitigate | The `is_prunable` predicate (Task 1) returns `false` for any status != `Exited`. If the `status` field fails to deserialize, `list_sessions()` already errors out that record (pre-existing behavior in `session.rs::load_session_file`). Predicate is fail-closed: unknown status → not prunable. |
| T-19-04-02 | Tampering (symlink escape) | A session file in `~/.nono/sessions/` is a symlink pointing to `/etc/passwd` or similar | mitigate | Both `run_prune` (Task 2) and `auto_prune_if_needed` (Task 3) use `std::fs::symlink_metadata(&path).file_type().is_symlink()` to refuse deletion of any session file that is a symlink. `remove_file` would follow a symlink and delete its target; `symlink_metadata` does NOT follow and gives us the right information. Additionally, `run_prune` canonicalizes the target and asserts it starts_with the sessions dir (defense in depth). |
| T-19-04-03 | Tampering | Path traversal via crafted session filename (`../../etc/passwd`) | mitigate | Session files are only ever opened by name pattern `{session_id}.json` joined to `sessions_dir()`. The filename itself comes from trusted listing, not from user input. `session_id` is generated internally (hex-encoded random). `Path::join` of a hex string cannot escape the parent dir. For defense in depth, the `canonicalize + starts_with` check in `run_prune` rejects any resolved path that escapes. |
| T-19-04-04 | Tampering | Future-dated `started_epoch` makes an exited session appear ancient | accept | `is_prunable` uses `saturating_sub(now - started)`, which returns 0 for `started > now`. With inclusive `>=` boundary: `0 >= retention_secs` is false when `retention_secs > 0` → not prunable. Fail-closed. Test `is_prunable_future_started_epoch_fails_closed` guards this. |
| T-19-04-05 | Denial of Service | Background auto-prune thread panics / deadlocks / runs forever | mitigate | The thread body is bounded (iterates a fixed list of IDs collected at spawn time). All error paths use `debug!` + `continue` — no panics or `.unwrap()`. The thread is NEVER joined, so a hung thread only leaks a single OS thread (bounded resource). The next `nono ps` will collect a fresh stale list and spawn a new thread; there is no cumulative thread leak because each `ps` invocation spawns at most one. |
| T-19-04-06 | Information Disclosure | `--dry-run` output reveals session IDs and timestamps | accept | `nono ps` and `nono inspect` already surface this information. Prune does not add new disclosure surface. Running `nono prune` is gated by `reject_if_sandboxed`, so a sandboxed agent cannot use this to enumerate parent-host sessions. |
| T-19-04-07 | Elevation of Privilege | Sandboxed process calls `nono ps` → triggers `auto_prune_if_needed` → deletes host supervisor's arbitrary session files | mitigate | `auto_prune_if_needed` (Task 3) has a STRUCTURAL early return as its FIRST statement: `if std::env::var_os("NONO_CAP_FILE").is_some() { return; }`. This matches the canonical sandbox-detection pattern used by `reject_if_sandboxed` throughout the CLI (session_commands.rs:21, session_commands_windows.rs:12). The guard is silent (no error) because `nono ps` itself is legal inside a sandbox — only the background-deletion side effect is forbidden. Regression-protected by the `auto_prune_is_noop_when_sandboxed` test (env-guarded via `lock_env` + `EnvVarGuard` per CLAUDE.md), which is mirrored to the Windows build. Acceptance criteria in Task 3 grep for `NONO_CAP_FILE` occurrence count ≥ 2 in both `session_commands.rs` and `session_commands_windows.rs` (pre-existing `reject_if_sandboxed` + new `auto_prune_if_needed`). |
| T-19-04-08 | Tampering (TOCTOU) | Supervisor may still be WRITING to a session file when prune deletes it | accept-with-guard | Only `Status: Exited` sessions are prunable. By contract, the supervisor sets `status = Exited` as its LAST write (see `SessionStatus::Exited` transitions in `session.rs` `finalize_record`). Reading a session file just after that final write and then deleting it is safe. If a supervisor crashes mid-write leaving a partial JSON, `load_session_file` errors and the record is excluded from `list_sessions()` → not considered for prune at all. |
| T-19-04-09 | Tampering | Clock set backward (Windows NTP skew) makes `now_epoch - started_epoch` negative | mitigate | `saturating_sub` in `is_prunable` handles this case (returns 0 → not prunable under inclusive `>=` when `retention_secs > 0`). Covered by `is_prunable_future_started_epoch_fails_closed` test. No exception path to "prune anyway". |

**High-severity threats:** T-19-04-02 (symlink escape) and T-19-04-07 (EoP via sandboxed ps). Both are mitigated in-plan: symlink_metadata check for T-19-04-02, and a structural NONO_CAP_FILE early return (FIRST statement of `auto_prune_if_needed`, grep-verified in Task 3 acceptance criteria) for T-19-04-07. The T-19-04-07 mitigation is additionally covered by the `auto_prune_is_noop_when_sandboxed` regression test, mirrored to the Windows build.
