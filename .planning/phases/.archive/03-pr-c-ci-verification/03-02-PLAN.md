1. In `scripts/windows-test-harness.ps1`, modify the "security" case in the switch block (currently line 156-158) to gate WFP tests on `$env:NONO_CI_HAS_WFP` per D-05.

   Replace the current unconditional:
   ```
   "security" {
       Invoke-TestList -LogFile "windows-security.log" -Tests $securityTests
   }
   ```

   With a split that runs non-WFP tests unconditionally and WFP tests only when gated:
   ```powershell
   "security" {
       $wfpFilters = @(
           "windows_run_block_net_blocks_probe_connection",
           "windows_run_block_net_cleans_up_promoted_wfp_filters_after_exit"
       )
       $nonWfpTests = $securityTests | Where-Object { $_.Filter -notin $wfpFilters }
       $wfpTests = $securityTests | Where-Object { $_.Filter -in $wfpFilters }

       Invoke-TestList -LogFile "windows-security.log" -Tests $nonWfpTests

       if ($env:NONO_CI_HAS_WFP -eq 'true') {
           Invoke-TestList -LogFile "windows-security.log" -Tests $wfpTests
       } else {
           $msg = "SKIPPED: WFP tests require elevated runner (NONO_CI_HAS_WFP not set)"
           $msg | Tee-Object -FilePath (Join-Path $LogDir "windows-security.log") -Append
       }
   }
   ```

   IMPORTANT: The $securityTests array declaration (lines 103-114) stays UNCHANGED — the WFP entries remain discoverable. The gate is at the call site only. The skip branch must NOT throw or exit non-zero ($ErrorActionPreference = "Stop" is active). Using Tee-Object for output is safe.

2. In `.github/workflows/ci.yml`, add a job-level `env:` block to the `windows-security` job per D-04. Insert after the `runs-on: windows-latest` line (line 240), before `steps:` (line 241):

   ```yaml
       env:
         NONO_CI_HAS_WFP: true   # windows-latest runners run as Administrator
   ```

   This follows the same YAML pattern as the global env block (lines 12-14) but scoped to the job. Hardcoded `true` is correct because windows-latest GitHub Actions runners are always Administrator. No secrets or expressions needed.
