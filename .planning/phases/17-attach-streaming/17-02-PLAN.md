```markdown
## Limitations on Windows detached sessions

**No terminal resize.** Detached sessions on Windows (`nono run --detached ...`) use anonymous-pipe stdio for child I/O — they do NOT allocate a Windows pseudo-console (ConPTY). This is a structural consequence of the Phase 15 `0xC0000142` fix and cannot be relaxed without re-introducing the loader-failure cascade.

The practical effect: `ResizePseudoConsole` cannot be invoked on the detached path, so terminal-resize events from your local terminal are NOT propagated to the child process. Most CLI tools work correctly under the default 80×24 geometry, but full-screen TUI applications (`vim`, `htop`, `less` in alternate-screen mode) may render incorrectly.

**Workarounds:**
- For full TUI fidelity on Windows, use `nono shell` (interactive PTY-attached shell) or non-detached `nono run -- <command>` (foreground PTY).
- For tools that honor the `COLUMNS` / `LINES` environment variables (rare on Windows), set them in your invocation: `nono run --detached -- cmd /c "set COLUMNS=120 & set LINES=40 & ...".`

This limitation is tracked under requirement ATCH-01 (see `.planning/REQUIREMENTS.md`) with the resize-propagation acceptance criterion (#3) explicitly downgraded per Phase 17 design decision D-07 (`.planning/phases/17-attach-streaming/17-CONTEXT.md`).
```
