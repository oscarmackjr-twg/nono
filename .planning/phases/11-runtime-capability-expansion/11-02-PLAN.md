<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sandboxed child → supervisor pipe | Untrusted CapabilityRequest messages arrive from a Low Integrity child process |
| terminal input → approval backend | User's y/N response read from CONIN$ or /dev/tty |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-02-01 | Spoofing | TerminalApproval CONIN$ open | mitigate | CONIN$ open failure mapped to Denied (not a crash or silent allow); no other process can inject into the host's console input buffer |
| T-11-02-02 | Tampering | CONIN$ read-line input | accept | User types directly into a console owned by the host supervisor process; sandboxed child cannot write to CONIN$ (it has no console handle); risk accepted |
| T-11-02-03 | Denial of Service | blocking read on CONIN$ from background thread | accept | EOF on console close returns immediately from read_line; thread denies the request and exits cleanly; no server hang |
| T-11-02-04 | Information Disclosure | session_token in AuditEntry | mitigate | Handler clones request and zeroes session_token field before building AuditEntry; test_audit_token_redaction verifies the invariant |
| T-11-02-05 | Elevation of Privilege | token-mismatch bypass in tests | mitigate | test_handle_token_mismatch uses a panic-if-called backend to structurally prove the approval backend is never reached on mismatch |
</threat_model>
