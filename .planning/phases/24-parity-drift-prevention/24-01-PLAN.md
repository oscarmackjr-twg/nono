### Categorization lookup table (Pattern 5; ORDER IS LOAD-BEARING — audit must match before generic crates/nono/src/* fallback)

**Bash `categorize_file()` (paste verbatim into `check-upstream-drift.sh`):**
```bash
# Categorize a file path into one of: profile, policy, package, proxy, audit, other.
# Path-prefix matching, first-match-wins. Order is load-bearing — audit must match
# before any generic crates/nono/src/* fallback (no such fallback exists today, but
# the comment locks the invariant for future maintainers).
categorize_file() {
  local f="$1"
  case "$f" in
    crates/nono-cli/src/profile/*|crates/nono-cli/src/profile.rs|crates/nono-cli/data/profile-authoring-guide.md)
      echo profile ;;
    crates/nono-cli/src/policy.rs|crates/nono-cli/data/policy.json)
      echo policy ;;
    crates/nono-cli/src/package*|crates/nono-cli/src/package_cmd.rs|crates/nono/src/package*)
      echo package ;;
    crates/nono-proxy/*)
      echo proxy ;;
    crates/nono/src/audit/*|crates/nono/src/audit_attestation*|crates/nono-cli/src/audit*)
      echo audit ;;
    *)
      echo other ;;
  esac
}
```

### Per-commit `categories` field (D-06)

For each commit, accumulate categories from its `files_changed`. Use a deduplicated, sorted list so JSON output is deterministic across the twin:

**Bash:**
```bash

declare -A CAT_SET=()
for f in "${FILES[@]}"; do
  CAT_SET[$(categorize_file "$f")]=1
done
# Emit sorted unique list. Sort lexicographically: audit, other, package, policy, profile, proxy.
```
